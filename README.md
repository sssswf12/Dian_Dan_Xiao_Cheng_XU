# 软糖小食铺点单小程序

一个原生微信小程序点单项目，适合个人、小型聚会、工作室或家庭场景使用。项目不包含在线支付，顾客提交订单后由店主在管理页确认、制作和完成。

## 功能

- 可爱卡通风点单首页
- 菜品分类、搜索、加入购物车、数量加减
- 堂食/自取、桌号或姓名、备注
- 我的订单记录、取消待确认订单、再来一单
- 店主管理页：口令登录、查看订单、接单、标记做好、完成、取消、复制订单
- 默认本地存储可直接预览
- 配置微信云开发后可上线收单

## 目录

```text
config/index.js                 店铺名、云环境 ID、管理员口令
utils/menu.js                   菜单数据
utils/cart.js                   购物车本地存储
utils/order-service.js          订单服务，本地和云开发两种模式
pages/menu                      点单页
pages/cart                      购物车和提交订单
pages/orders                    我的订单
pages/admin                     店主管理
cloudfunctions/orderService     云函数订单服务
```

## 本地预览

1. 打开微信开发者工具。
2. 导入本目录：`C:\Users\Admin\Desktop\1\点单小程序`。
3. AppID 可以先使用测试号或自己的小程序 AppID。
4. 保持 `config/index.js` 里的 `CLOUD_ENV_ID` 为空，即可用本地存储完整预览。

本地模式的订单只保存在当前设备，适合开发预览和单机自用。

## 上线收单

1. 在微信开发者工具里开通云开发，创建云环境。
2. 把云环境 ID 填入 [config/index.js](config/index.js) 的 `CLOUD_ENV_ID`。
3. 如需本地预览店主管理页，可以修改 [config/index.js](config/index.js) 的 `ADMIN_PIN`；这个值只适合本地模式。
4. 在云开发控制台创建数据库集合：`orders`。
5. 右键 `cloudfunctions/orderService`，选择安装依赖并上传部署。
6. 在云函数环境变量中设置 `ADMIN_PIN`，这是线上店主管理页真正使用的口令。
7. 开发者工具中预览，确认顾客下单、店主管理页接单都正常。
8. 点击上传，进入微信公众平台提交审核和发布。

云开发模式下，顾客订单会写入云数据库；店主管理页通过云函数读取和更新订单状态。

## 修改菜单

编辑 [utils/menu.js](utils/menu.js) 中的 `RAW_MENU_ITEMS`。

金额使用“分”为单位，例如：

```js
price: 2600
```

页面会显示为 `¥26`。

## 修改店铺信息

编辑 [config/index.js](config/index.js)：

```js
const SHOP = {
  name: '软糖小食铺',
  subtitle: '今日也要好好吃饭',
  notice: '无在线支付，提交订单后由店主确认制作。',
  serviceNote: '适合家庭、工作室、小型聚会的自助点单。',
};
```

## 注意

- 项目不采集手机号、地址等敏感个人信息。
- 线上店主管理口令必须配置在云函数环境变量中，不要把真实口令写在前端配置里。
- 店主管理口令适合个人和轻量场景；如果后续开放给更多用户，建议改成基于 OpenID 的管理员白名单。
- 如果暂时不需要多人实时收单，可以不配置云开发，继续使用本地模式。
