# 家庭点单小程序

一个家庭内部使用的原生微信小程序点单项目。项目不接入微信支付，不显示价格，不做收银，只用于家庭成员提交饮品、菜品、甜品、水果等需求，管理员在管理端查看和确认。

详细需求以 [docs/家庭点单小程序需求说明.md](docs/家庭点单小程序需求说明.md) 为准。

## 当前功能

- 统一登录页：管理员输入 `admin / 040815` 进入管理端，普通成员输入管理员新增的账号进入用户端。
- 点单页：菜品、饮品、甜品、水果、其他分类，支持搜索和数量加减。
- 确认下单页：查看已选内容，填写方式、位置、备注，点击确认下单。
- 我的订单：只显示当前成员自己的订单，支持取消待确认订单和再来一单。
- 管理端：管理员账号密码登录，查看全部订单，确认、完成、取消、复制订单。
- 菜单管理：管理员可新增、编辑、上架、下架菜单内容。
- 菜单图片：内置 49 张本地手绘 SVG 图样，编辑页用固定高度滚动区域选择。
- 图片上传：管理员可从相册或拍照上传菜单图片。
- 成员管理：管理员可新增、编辑、启用、停用家庭成员账号。
- 普通用户端不展示管理入口，管理页保留为独立页面。
- 登录后使用底部导航。
- 本地预览：不配置云环境也可以在单设备上完整预览。
- 云开发：配置云环境后支持多设备同步订单、菜单和成员账号。

## 本地预览

1. 打开微信开发者工具。
2. 导入本目录：`C:\Users\Admin\Desktop\1\点单小程序`。
3. AppID 可以先使用测试号或自己的小程序 AppID。
4. 保持 `config/index.js` 的 `CLOUD_ENV_ID` 为空。
5. 使用默认管理员账号 `admin`、密码 `040815` 进入管理端。

默认没有普通用户账号，进入管理端后在“成员”里新增。

本地模式的数据只保存在当前设备，适合开发预览和单机演示。

本地模式下上传的菜单图片使用微信临时路径，只适合当前设备预览。真实多设备使用时应开启云开发，让上传图片保存到云存储。

## 云开发上线

1. 在微信开发者工具里开通云开发，创建云环境。
2. 把云环境 ID 填入 [config/index.js](config/index.js) 的 `CLOUD_ENV_ID`。
3. 在云数据库创建集合：`orders`、`menuItems`、`users`。
4. 右键 `cloudfunctions/orderService`，安装依赖并上传部署。
5. 在云函数环境变量中设置 `ADMIN_PIN=040815`。
6. 预览小程序，测试账号登录、下单、管理端确认、菜单管理、成员管理。
7. 上传代码，进入微信公众平台提交审核和发布。

云开发模式下，菜单编辑页上传图片会通过 `wx.cloud.uploadFile` 保存到云存储的 `menu-images/` 目录。

## 常用修改

### 修改标题和文案

编辑 [config/index.js](config/index.js)：

```js
const SHOP = {
  name: '家庭点单',
  subtitle: '今天想吃什么',
  notice: '家庭内部点单，不需要支付，提交后由管理员确认。',
  serviceNote: '适合饮品、菜品、甜品、水果和日常需求。',
};
```

### 添加成员账号

默认不预置普通成员。进入管理端，在“成员”里新增账号、密码、名称和备注。

### 修改默认菜单

编辑 [utils/menu.js](utils/menu.js) 的 `RAW_MENU_ITEMS`。上线后也可以直接在管理端维护菜单。

### 修改菜单图片

内置图样放在 [assets/menu](assets/menu) 目录，当前共 49 张 SVG。菜单编辑页的“选择图片”是固定高度滚动选择区，不会因为图样数量增加而撑高页面。

如果要给某个菜单上传真实图片，进入管理端“菜单”，点击新增或编辑内容，再点击“上传图片”。云开发模式下图片会保存到云存储；本地预览模式下只保存当前设备可用的本地路径。

## 注意

- 项目不包含支付功能，也不显示价格。
- 云开发模式下，管理员密码必须同步配置到云函数环境变量 `ADMIN_PIN` 中。
- 多设备真实使用时建议开启云开发，否则订单、菜单、成员和上传图片都只适合单设备预览。
- 成员账号适合家庭内部区分身份，不是强安全登录。
- 如果后续要加强权限，可以把成员账号绑定微信 OpenID。
