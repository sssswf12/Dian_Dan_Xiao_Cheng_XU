# 家庭点单小程序需求说明

## 1. 项目定位

本项目是家庭内部使用的微信点单小程序。它不是商业收银系统，不接入微信支付，不显示价格，不产生付款流程。

小程序只解决三件事：

- 用户进入小程序后先看到统一登录页；管理员用 `admin / 040815` 登录，普通成员用管理员新增的账号登录。
- 成员确认下单后，订单进入管理员端。
- 管理员查看订单、确认订单、完成订单，并维护菜单和家庭成员账号。

## 2. 角色

### 2.1 普通成员

普通成员可以：

- 输入自己的账号和密码进入小程序。
- 浏览已上架的菜单内容。
- 按分类搜索和点选内容。
- 提交订单。
- 查看自己的订单记录。
- 取消尚未确认的订单。
- 将历史订单重新放回购物车。

普通成员不可以：

- 修改菜单。
- 查看所有成员订单。
- 处理订单状态。
- 修改成员账号。

### 2.2 管理员

管理员只有一个默认账号，通过账号 `admin` 和密码 `040815` 进入管理端。

管理员可以：

- 查看所有成员提交的订单。
- 按订单状态筛选订单。
- 将订单从待确认改为已确认。
- 将订单从已确认改为已完成。
- 取消待确认订单。
- 复制订单内容。
- 新增、编辑、上架、下架菜单内容。
- 新增、编辑、启用、停用家庭成员账号。

管理员账号：

- 账号：`admin`
- 密码：`040815`
- 云开发上线时，云函数环境变量 `ADMIN_PIN` 必须设置为 `040815`。

## 3. 页面规划

### 3.1 登录 / 我

路径：`pages/account/account`

用途：

- 输入账号和密码登录。
- 管理员账号进入管理端。
- 普通成员账号进入点单页。
- 普通成员登录后作为“我”页面，展示当前成员账号。
- 普通成员可在“我”里退出登录。
- 普通用户端不展示管理端入口。

登录前不展示业务导航。普通成员登录后底部导航为：点单、确认、订单、我。

### 3.2 点单页

路径：`pages/menu/menu`

用途：

- 展示点单标题。
- 展示分类：全部、菜品、饮品、甜品、水果、其他。
- 支持关键词搜索。
- 展示已上架菜单。
- 支持加入购物车和数量加减。

页面不显示价格。

### 3.3 确认页

路径：`pages/cart/cart`

用途：

- 查看已选择内容。
- 调整数量。
- 清空购物车。
- 填写用餐方式、位置和备注。
- 点击“确认下单”提交订单。

订单提交前必须满足：

- 已登录成员账号。
- 已选内容里至少有一个内容。

位置和备注为可选项。

### 3.4 我的订单页

路径：`pages/orders/orders`

用途：

- 只展示当前成员自己的订单。
- 查看订单状态。
- 取消待确认订单。
- 再来一单。

### 3.5 管理端

路径：`pages/admin/admin`

用途：

- 订单管理。
- 菜单管理。
- 成员管理。
- 管理员账号信息和退出登录。

管理端只允许管理员账号进入，普通成员不能看到管理入口。管理端底部导航为：订单、菜单、成员、我。

### 3.6 菜单编辑页

路径：`pages/menu-edit/menu-edit`

用途：

- 新增菜单内容。
- 编辑已有菜单内容。
- 选择分类、单位、标签、排序、描述和上架状态。
- “选择图片”区域以固定高度滚动面板展示本地手绘 SVG 内置图样。
- 支持从相册或拍照上传图片。
- 云开发模式下，上传图片保存为云存储文件；本地预览模式下保存为当前设备可预览的本地路径。

## 4. 菜单分类

默认分类：

- 菜品：日常主食、热菜、小吃等。
- 饮品：水、茶、果汁、奶饮等。
- 甜品：布丁、蛋糕、小饼等。
- 水果：切果、水果杯等。
- 其他：临时需要添加的内容。

菜单字段：

```text
id             唯一编号
category       分类编号
categoryName   分类名称
name           名称
desc           描述
unit           单位
tag            标签，可为空
enabled        是否上架
sort           排序值
art            前端展示样式
themeBg        卡片背景色
themeAccent    点缀色
imageUrl       图片路径，可为空
```

菜单不包含价格字段。

菜单图片优先使用 `assets/menu` 下的本地 SVG 图样，当前内置 49 张。管理员也可以上传自己的图片替换内置图样。

## 5. 成员账号

成员账号字段：

```text
id        唯一编号
account   登录账号
password  登录密码
name      成员名称
note      备注，可为空
enabled   是否启用
```

普通成员账号登录适合家庭内部使用。它的作用是区分是谁下单、谁查看自己的订单。

如果后续需要更强的安全性，可以把成员账号和微信 OpenID 绑定。

## 6. 订单流程

订单状态：

```text
pending    待确认
confirmed  已确认
completed  已完成
cancelled  已取消
```

流转规则：

```text
成员下单 -> pending 待确认
管理员确认 -> confirmed 已确认
管理员完成 -> completed 已完成
成员或管理员取消 -> cancelled 已取消
```

取消规则：

- 成员只能取消自己的 `pending` 订单。
- 管理员可以取消 `pending` 订单。
- 已确认或已完成订单不允许普通成员取消。

订单字段：

```text
id / recordId  订单记录编号
orderNo        展示用订单号
status         订单状态
userId         下单成员编号
userName       下单成员名称
diningType     用餐方式
location       位置，可为空
remark         备注，可为空
items          订单内容
totalCount     总数量
createdAtMs    创建时间
updatedAtMs    更新时间
```

订单不包含支付状态，不包含金额。

## 7. 数据存储

### 7.1 本地预览模式

`config/index.js` 中的 `CLOUD_ENV_ID` 为空时，使用本地存储。

适合：

- 微信开发者工具预览。
- 单设备演示。
- 开发调试。

限制：

- 数据只存在当前设备。
- 多个手机之间不能同步订单、菜单和成员账号。
- 上传图片保存为当前设备可预览的本地路径，不适合多设备长期使用。

### 7.2 云开发模式

填写 `CLOUD_ENV_ID` 后，使用云函数和云数据库。

需要云数据库集合：

```text
orders     订单
menuItems  菜单
users      家庭成员账号
```

需要部署云函数：

```text
cloudfunctions/orderService
```

需要云函数环境变量：

```text
ADMIN_PIN=040815
```

云开发模式适合真实家庭多设备使用。

菜单编辑页上传的图片保存到云存储 `menu-images/` 目录，菜单数据保存云文件 ID。

## 8. 开发顺序

后续开发按以下顺序执行：

1. 先维护本需求文档。
2. 调整配置、菜单、成员账号和订单数据结构。
3. 改普通成员端：账号登录、点单、下单、我的订单。
4. 改管理端：订单处理、菜单管理、成员管理。
5. 扩展云函数，支持云端菜单和成员账号。
6. 最后检查文档、页面文案和上线配置。
