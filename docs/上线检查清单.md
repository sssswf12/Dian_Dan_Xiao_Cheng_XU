# 上线检查清单

## 必改项

- 在微信公众平台创建自己的小程序并拿到 AppID。
- 在微信开发者工具里把项目 AppID 从测试号改成自己的 AppID。
- 修改 `config/index.js` 里的 `SHOP` 店铺信息。
- 修改 `utils/menu.js` 里的菜单名称、价格、描述和分类。
- 如果要多人实时收单，创建云开发环境，并把环境 ID 填到 `config/index.js` 的 `CLOUD_ENV_ID`。
- 云开发模式下，在云函数 `orderService` 的环境变量里设置 `ADMIN_PIN`。

## 云开发步骤

1. 微信开发者工具打开项目。
2. 点击“云开发”，创建环境。
3. 数据库中新建集合：`orders`。
4. 右键 `cloudfunctions/orderService`，选择“安装依赖”。
5. 右键 `cloudfunctions/orderService`，选择“上传并部署：云端安装依赖”。
6. 到云函数配置里添加环境变量：`ADMIN_PIN=你的店主口令`。
7. 预览小程序，顾客提交订单。
8. 店主页输入口令，确认能看到并处理订单。

## 提审前自测

- 点单页分类切换正常。
- 搜索菜品正常。
- 加入购物车、加减数量、清空购物车正常。
- 未填写桌号或姓名时不能提交订单。
- 提交订单后跳转到订单页。
- 待确认订单可以取消。
- 再来一单能把历史订单放回购物车。
- 店主管理页口令错误时不能进入。
- 店主管理页可以接单、标记做好、完成订单。
- 复制订单内容能粘贴到微信聊天或备忘录。

## 审核注意

- 这个项目没有在线支付，不需要配置微信支付。
- 如果用于真实食品售卖，具体经营资质、类目和审核材料以微信公众平台当前要求为准。
- 个人自用、家庭或内部小范围使用时，建议在页面文案中保持“提交订单后由店主确认制作”这类说明。
