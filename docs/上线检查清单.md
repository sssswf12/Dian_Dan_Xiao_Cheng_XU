# 上线检查清单

## 必改项

- 在微信公众平台创建自己的小程序并拿到 AppID。
- 在微信开发者工具里把项目 AppID 从测试号改成自己的 AppID。
- 修改 `config/index.js` 里的 `SHOP` 文案。
- 默认没有普通成员账号，首次进入后用管理员账号新增成员。
- 修改 `utils/menu.js` 里的默认菜单，或上线后在管理端维护菜单。
- 如需多设备同步，创建云开发环境，并把环境 ID 填到 `config/index.js` 的 `CLOUD_ENV_ID`。
- 云开发模式下，在云函数 `orderService` 的环境变量里设置 `ADMIN_PIN=040815`。

## 云开发步骤

1. 微信开发者工具打开项目。
2. 点击“云开发”，创建环境。
3. 数据库中新建集合：`orders`、`menuItems`、`users`。
4. 右键 `cloudfunctions/orderService`，选择“安装依赖”。
5. 右键 `cloudfunctions/orderService`，选择“上传并部署：云端安装依赖”。
6. 到云函数配置里添加环境变量：`ADMIN_PIN=040815`。
7. 预览小程序，使用 `admin / 040815` 登录管理端。
8. 在管理端新增一个普通成员账号。
9. 使用普通成员账号登录并提交订单。
10. 在管理端测试新增菜单、下架菜单、新增成员、停用成员。

## 提审前自测

- 账号登录页能用正确账号和密码登录。
- `admin / 040815` 登录后进入管理端。
- 退出登录后不能继续用原身份下单。
- 登录不同账号后，“我的订单”只显示当前账号自己的订单。
- 点单页分类切换正常。
- 搜索内容正常。
- 加入购物车、加减数量、清空购物车正常。
- 未登录账号时不能提交订单。
- 确认下单后能跳转到订单页。
- 待确认订单可以取消。
- 再来一单能把历史订单放回购物车。
- 管理端口令错误时不能进入。
- 管理端可以确认、完成、取消订单。
- 管理端可以复制订单内容。
- 管理端可以新增、编辑、上架、下架菜单。
- 管理端可以新增、编辑、启用、停用成员。
- 普通用户端不显示管理入口。

## 审核注意

- 这个项目没有在线支付，不需要配置微信支付。
- 页面文案应保持“家庭内部点单、提交后由管理员确认”的说明。
- 如果用于真实食品经营，经营资质、类目和审核材料以微信公众平台当前要求为准。
