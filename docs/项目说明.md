# 点单小程序项目说明

## 项目定位

这是一个家庭内部使用的微信点单小程序。项目不接入在线支付，不显示价格，只用于家庭成员提交吃喝和日常需求，管理员在管理端确认和处理。

需求边界以 [家庭点单小程序需求说明.md](家庭点单小程序需求说明.md) 为准。

## 核心功能

- 统一登录：管理员输入 `admin / 040815` 进入管理端，普通成员输入管理员新增的账号进入用户端。
- 点单首页：展示分类、搜索、菜单卡片、加入购物车和数量加减。
- 确认下单：确认已选内容，填写用餐方式、位置、备注，点击确认下单。
- 我的订单：查看当前成员自己的历史订单、状态、取消待确认订单、再来一单。
- 管理端：管理员账号密码登录，查看全部订单，确认、完成、取消、复制订单。
- 菜单管理：管理员新增、编辑、上架、下架菜品、饮品、甜品、水果和其他内容。
- 菜单图片：内置 49 张本地手绘 SVG 图样，菜单编辑页用固定高度滚动区域选择。
- 图片上传：管理员可以从相册或拍照上传菜单图片。
- 成员管理：管理员新增、编辑、启用、停用家庭成员账号。
- 普通用户端不展示管理入口，管理页作为独立页面保留。
- 登录后使用底部导航，不在页面顶部放业务导航。
- 云开发服务：上线后通过云函数和云数据库同步订单、菜单和成员账号。
- 本地预览模式：不配置云环境时，可用本地存储跑完整流程。

  订单服务适配层。未配置云环境时走本地存储；配置云环境后调用云函数。

  账号登录页。

pages/menu
  点单首页。

  管理端：订单、菜单、成员、我。

## 订单状态流转

订单默认状态是 `pending`，表示待确认。

```text
pending 待确认
  -> confirmed 已确认
  -> completed 已完成
```

订单也可以进入：

```text
cancelled 已取消
```

成员只能取消自己的 `pending` 订单。订单确认后，普通成员不能取消。

## 本地模式和云开发模式

### 本地模式

`config/index.js` 中的 `CLOUD_ENV_ID` 为空时，小程序使用本地存储。

适合：

- 微信开发者工具预览。
- 单设备演示。
- 开发调试。

限制：

- 订单、菜单、成员账号只保存在当前设备。
- 其他设备看不到这些数据。
- 上传图片保存为当前设备可预览的本地临时路径，不适合多设备长期使用。

### 云开发模式

填写 `CLOUD_ENV_ID` 后，小程序会调用 `orderService` 云函数。

适合：

- 家庭多设备使用。
- 多个成员提交订单。
- 管理员统一查看和处理订单。
- 管理员维护菜单和成员账号。
- 菜单上传图片通过云存储保存，路径位于 `menu-images/`。

云开发模式需要：

- 开通微信云开发。
- 创建 `orders`、`menuItems`、`users` 数据库集合。
- 上传部署 `cloudfunctions/orderService`。
- 在云函数环境变量中配置 `ADMIN_PIN=040815`。

## 菜单图片规则

菜单内容有一个 `imageUrl` 字段。默认菜单和管理员新增菜单会优先使用 `assets/menu` 下的 SVG 图样。

管理员在菜单编辑页可以：

- 在固定高度滚动区选择内置 SVG 图样。
- 点击“上传图片”从相册或相机选择图片。
- 点击“恢复图样”改回当前分类对应的默认内置图样。

本地模式上传图片只适合开发者工具和当前设备预览。云开发模式会把图片上传到云存储，保存返回的云文件 ID。
