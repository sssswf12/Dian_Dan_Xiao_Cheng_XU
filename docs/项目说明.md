# 点单小程序项目说明

## 项目定位

这是一个面向个人、小型聚会、工作室或家庭场景的微信点单小程序。项目不接入在线支付，顾客只负责选菜和提交订单，店主在管理页确认订单、制作、标记可取餐和完成。

整体风格是可爱、卡通、轻量化，适合个人自用或小范围内部使用。

## 核心功能

- 点单首页：展示店铺信息、菜品分类、搜索、菜品卡片、加入购物车。
- 购物车：查看已选菜品、修改数量、填写堂食或自取信息、提交订单。
- 我的订单：查看历史订单、查看订单状态、取消待确认订单、再来一单。
- 店主管理：输入口令进入，查看全部订单，接单、取消、标记做好、完成订单，并支持复制订单内容。
- 云开发订单服务：上线后通过云函数和云数据库保存订单，支持多人提交和店主统一处理。
- 本地预览模式：不配置云环境时，可以直接在微信开发者工具里用本地存储跑完整流程。

## 订单状态流转

订单默认状态是 `pending`，表示待确认。

店主处理流程：

```text
pending 待确认
  -> accepted 制作中
  -> ready 可取餐
  -> completed 已完成
```

订单也可以进入：

```text
cancelled 已取消
```

顾客只能取消 `pending` 状态的订单。订单开始处理后，取消操作会被拒绝。

## 本地模式和云开发模式

### 本地模式

`config/index.js` 中的 `CLOUD_ENV_ID` 为空时，小程序使用本地存储。

适合：

- 微信开发者工具预览
- 单设备个人自用
- 功能演示

限制：

- 订单只保存在当前设备。
- 其他设备看不到这些订单。

### 云开发模式

填写 `CLOUD_ENV_ID` 后，小程序会调用 `orderService` 云函数。

适合：

- 真实上线收单
- 多个顾客提交订单
- 店主在一个管理页统一查看和处理订单

云开发模式需要：

- 开通微信云开发
- 创建 `orders` 数据库集合
- 上传部署 `cloudfunctions/orderService`
- 在云函数环境变量中配置 `ADMIN_PIN`

## 常用修改位置

### 修改店铺名称和文案

文件：[config/index.js](../config/index.js)

```js
const SHOP = {
  name: '软糖小食铺',
  subtitle: '今日也要好好吃饭',
  notice: '无在线支付，提交订单后由店主确认制作。',
  serviceNote: '适合家庭、工作室、小型聚会的自助点单。',
};
```

### 修改菜单

文件：[utils/menu.js](../utils/menu.js)

菜品价格使用“分”为单位：

```js
price: 2600
```

页面显示为：

```text
¥26
```

### 修改本地预览店主口令

文件：[config/index.js](../config/index.js)

```js
const ADMIN_PIN = '2468';
```

注意：这个口令只适合本地预览。线上真实口令必须放在云函数环境变量 `ADMIN_PIN` 中。

## 上线流程概览

1. 用微信开发者工具导入项目目录。
2. 把 `project.config.json` 里的测试 AppID 改成自己的小程序 AppID，或在开发者工具导入时选择自己的 AppID。
3. 修改店铺信息和菜单。
4. 开通云开发并创建云环境。
5. 把云环境 ID 填到 `config/index.js` 的 `CLOUD_ENV_ID`。
6. 在云数据库创建集合 `orders`。
7. 部署 `cloudfunctions/orderService` 云函数。
8. 给云函数配置环境变量 `ADMIN_PIN`。
9. 在开发者工具预览，测试顾客下单和店主接单。
10. 上传代码，进入微信公众平台提交审核。

## 安全说明

- 项目不包含微信支付，不会产生线上扣款行为。
- 前端代码里的口令不能作为正式安全措施，因为小程序前端包可能被查看。
- 线上店主口令必须配置在云函数环境变量中。
- 如果后续要多人共同管理，建议把店主管理改为 OpenID 白名单。
